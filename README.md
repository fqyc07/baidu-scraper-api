## 百度搜索结果为什么这么难抓

百度的反爬机制比大多数人想象的复杂。它不只看 IP 频率，还会检测浏览器环境、Cookie 状态、请求头一致性。你用 requests 库直接发请求，大概率第三次就会触发验证码。用 Selenium 模拟浏览器？资源消耗大不说，百度照样能通过 WebDriver 特征识别你。

我自己踩过的坑：用了一个月免费代理池，抓取 500 个关键词的百度排名数据，实际成功返回的不到一半。剩下的要么是空页面，要么是验证码页。

ScraperAPI 的做法是在服务端维护一个庞大的住宅代理网络，每次请求自动轮换 IP、自动处理 Cookie 和指纹。对百度这种目标，它还支持设置 `country_code=cn` 来走中国节点，成功率明显高于通用代理。

## 实际接入百度抓取只需要几行代码

接入方式很直白。你注册拿到 API Key 之后，把百度搜索的 URL 编码后拼到请求里就行：

```
http://api.scraperapi.com?api_key=YOUR_KEY&url=https://www.baidu.com/s?wd=关键词&country_code=cn
```

返回的就是完整的百度搜索结果 HTML。你再用 BeautifulSoup 或者 lxml 解析排名数据，整个流程比自建代理池省心太多。

## 常见问题

### ScraperAPI 抓百度搜索结果合法吗？

ScraperAPI 抓取的是公开可访问的网页内容。百度搜索结果页本身是公开的，但具体使用场景是否合规取决于你所在地区的法律和你的用途。用于 SEO 排名监控、市场调研这类场景，通常没有问题。

### 免费版有什么限制？

5000 次请求/月，5 个并发线程。功能上和付费版一样，支持地理定位、JavaScript 渲染等。用完额度当月就停，不会自动扣费。

### 百度抓取消耗的 credits 和普通网页一样吗？

不一样。百度属于受保护目标，每次请求会消耗额外的 API credits（通常是普通请求的 5-10 倍）。建议在 Dashboard 里关注实际消耗速度，按需选择套餐。

### 支持抓取百度的哪些页面？

搜索结果页、百度知道、百度百科、百度贴吧等公开页面都可以。只要是浏览器能正常访问的百度页面，理论上都能通过 ScraperAPI 抓取。

### 抓取失败怎么办？

ScraperAPI 对失败请求不计费。如果返回非 200 状态码，不会扣你的额度。建议在代码里加重试机制，设置 `retry=3` 参数即可。
